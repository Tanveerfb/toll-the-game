# Status ‚Äî 2026-07-12

Living snapshot. History of the resurrection audit is in git (`docs/STATUS.md` @ `c3040f7`).

## Working (implemented, tested, browser-verified)

- **Battle engine** ‚Äî full phase state machine, 3-actions-per-turn enemy AI (random living field enemy each action, decisions from live state), win/loss detection.
- **Rank system** ‚Äî card rank drives damage multiplier, `*Ranked` mechanic values, and `aoeRanked` activation; flat mechanic values stay flat; ultimates rank-immune.
- **Deck (7DS GC rules)** ‚Äî hand never resets; pure-random one-at-a-time refill with auto-merge on adjacent identical cards (+1 ult gauge per merge); ult guaranteed only if gauge was full BEFORE the refill; deck locked outside `PlayerAction`; empty-hand auto-pass.
- **Enemy hidden deck (2026-07-18)** ‚Äî the enemy side now runs the same 7DS GC deck as the player, headless: a shared enemy hand seeded at battle start, RNG-refilled to the same capacity (`[0,4,5,7,8]` by field count) each enemy turn, auto-merging adjacent identical cards up a rank (each merge grants that enemy +1 ult gauge). The AI plays ONLY from this hand (`getAIMove(..., hand)`) by the existing priority tiers, so enemy skills finally use their merged **rank** and enemy ults come through the gauge/deck ‚Äî RNG-fair like the player. Merge/refill logic extracted to `lib/game/deck.ts` and shared by both sides (player `gameStore` refactored onto it, behavior unchanged). Telegraph ("NEXT" cards above enemies) deferred to the battle-arena polish pass. `Action.cardId` lets the loop consume the played card.
- **Molvarr boss engine (2026-07-18, partial)** ‚Äî building the first multi-phase world boss. DONE: per-character ult-gauge cap (`ultGaugeMax`, default 5 / Molvarr 10; `lib/game/ultGauge.ts`, every hardcoded 5 replaced); **Corrosion** mechanic (uncapped-stacking DoT = % of target MAX HP per stack/turn, ticks at victim turn-end; `combat.ts`+`tick.ts`); **phase system** (`lib/game/phases.ts` + `CharacterPhase`/`phases[]`/`phaseIndex` + schema) ‚Äî a boss transitions to its next phase at HP 0 (fresh bar, boss buffs/debuffs/gauge/per-phase-state reset; player state + global turn counter + Corrosion-on-players persist), wired into the battle loop (player-action + DoT-tick paths, before victory; enemy hand redraws from new-phase skills); **CC-immunity** (`ccImmune` blocks stun). PENDING: 5 boss passives (SP-timer / turn-10 spike+drain / debuff-count‚ÜíATK / corrosion-spread / +30%-vs-corroded) which share one new sub-system (per-phase turn-hook + SP-passive data + per-phase `spSkill` slot + multi-passive activation), then the Molvarr kit data. Full spec: `docs/design/BOSS_MOLVARR.md`.
- **Kit Lab (2026-07-18, dev-only `/kit-lab`)** ‚Äî GUI to author/simulate/save character kits, reusing the real engine (mechanics browser over all 42 types, live `KitDetails` preview + auto Dokkan descriptions, `damagePreview` sim + Test-in-Battle, roster-median balance flags via `lib/game/balance.ts`, Zod-validated direct save to `data/characters/*.json`, new-mechanic briefs ‚Üí `docs/proposed-mechanics/`). 404s in prod. Design spec: `docs/superpowers/specs/2026-07-18-kit-lab-design.md`.
- **Sub units** ‚Äî battle format 4v4 (all field) or 3v3 (4th member auto-sub); sub passive active from bench, no cards, untargetable; promoted at new-turn start after a teammate dies; lone subs auto-convert to field.
- **Character kits** ‚Äî 16 JSON kits incl.
